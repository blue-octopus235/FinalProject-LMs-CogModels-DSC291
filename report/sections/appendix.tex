\section{Reproducibility Details}
\label{app:repro}

% OWNER: Jackson Wilke (methods/infra per proposal)

\paragraph{Pipeline.} The LSTM pipeline is
\texttt{data\_utils} $\to$ \texttt{train} $\to$ checkpoint $\to$
\texttt{evaluate} $\to$ \texttt{results/eval\_results.csv} $\to$ \texttt{plots}.
\texttt{data\_utils} builds the shared vocabulary and the fixed 20K held-out
indices; \texttt{minimal\_pairs} and \texttt{verb\_flip} construct the leak-free
test pairs from the clean corpus. The RNNG pipeline
(\texttt{run\_rnng\_datahub.py}) parses the clean corpus once with the Berkeley
Neural Parser, swaps the verb terminal per noise condition, runs the
\texttt{rnng-pytorch} preprocessing and training, and scores the shared minimal
pairs by beam search.

\paragraph{Matched-subset comparison (H3).} Because the RNNG and LSTM cover
slightly different pair sets (Section~\ref{sec:metrics}), the cross-model
comparison is computed on their intersection:
\begin{enumerate}
  \item \texttt{run\_rnng\_datahub.py -{}-seeds 1 2 3} writes per-pair records
    (\texttt{results/rnng\_pairs\_*.csv}) and the indices each run could score.
  \item \texttt{src/match\_subset.py} forms the matched index set
    (RNNG-covered $\cap$ LSTM-in-vocabulary) and recomputes RNNG metrics on it.
  \item \texttt{src/evaluate.py -{}-restrict\_indices results/matched\_idx.json}
    re-scores each LSTM checkpoint on the same subset.
  \item \texttt{src/combined\_plots.py} produces the side-by-side figures, and
    \texttt{src/stats\_h3.py} computes the per-condition gap difference with its
    bootstrap confidence interval and $p$-value.
\end{enumerate}

\paragraph{Validity checks.} The RNNG run also emits a per-epoch validation
perplexity log (\texttt{results/rnng\_train\_ppl.csv}) to confirm convergence,
and a parser-quality summary plus a sample of parsed trees
(\texttt{results/rnng\_parse\_quality.json}, \texttt{rnng\_tree\_sample.txt}) for
hand-inspection, since parser quality on lowercased, anonymized tokens bounds the
RNNG's ceiling.
